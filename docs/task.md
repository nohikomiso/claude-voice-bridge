# 今後のタスクリスト (Tasks)

現在の `claude-input` はPoC（概念実証）の段階であり、コア機能である「クリップボード経由でのターミナルへのペースト（Enterなし）」のみが実装されています。

以下のタスクは、将来的に完全自動化およびDiscord連携を実現するためのロードマップです。

## フェーズ1: PoCの検証 (現在完了・要テスト)
- [x] プロジェクトと仮想環境(`uv`)のセットアップ
- [x] OSクリップボードへのテキストコピー処理 (`pyperclip`)
- [x] ペーストキー(`Shift+Insert`)の送信処理 (`pyautogui`)
- [ ] Windows Terminal上で `main.py` を実行し、手動でclaudeのウィンドウをアクティブにして正しくペーストされるか検証する。

## フェーズ2: 対象ウィンドウの自動アクティブ化
現在の「3秒待つ間に人間がクリックする」運用をなくし、プログラムが自動で `claude` が動いているターミナルを最前面に持ってくる処理を実装します。

- [ ] **Windows対応**: `pygetwindow` などのライブラリを使い、特定のタイトル（例: "claude"）を持つウィンドウを検索し、フォアグラウンドに引き上げる処理を `injector.py` に追加する。
- [ ] **Ubuntu(X11)対応**: `subprocess` 経由で `wmctrl` または `xdotool` コマンドを呼び出し、指定したターミナルウィンドウをアクティブにする処理を追加する。

## フェーズ3: バックグラウンド常駐化とWeb API/Discord連携
Discordからのメッセージを受け取るための口を作ります。

- [ ] ローカルで立ち上がる軽量なHTTPサーバー（例: FastAPI や Flask）を組み込み、外部からPOSTリクエストでテキストを受け取れるようにする。
- [ ] または、直接 `discord.py` を組み込み、特定のBotアカウント宛のメンションなどを監視して、受信したテキストを `TerminalInjector` に流し込む仕組みを構築する。

## フェーズ4: エッジケースへの対応
- [ ] 送信されたテキストが空だった場合のエラーハンドリング。
- [ ] 対象のターミナル（claude）が見つからなかった場合のリトライ処理や警告メッセージの表示。
- [ ] クリップボードを上書きしてしまうため、元のクリップボードのテキストを一時保存し、ペースト後に元に戻す親切な設計の検討。
