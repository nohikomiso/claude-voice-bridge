# 実装計画: claude-input-injector ツール (最小構成 PoC)

## Context
現在ユーザーはWindows (Windows Terminal) やUbuntu (X11) 上で `claude` CLI を起動して使用しています。将来的にDiscordから受け取ったメッセージを、既に起動している `claude` のプロンプト（入力待ち画面）に「Enterを押さずにペーストだけする」仕組みを構築したいと考えています。

既存の `ccdiscord` のようにCLIをサブプロセスとして完全にバックグラウンドで自動化するのではなく、今回は「人間の目の前で動いているCLI画面に対して、外部プログラムから文字列を注入する」という全く異なるアプローチ（OSのGUIエミュレーション）が必要です。

今回はその最終目標に向けた**最小単位の実証コード (PoC)** を作成します。現在の実行環境であるWindowsに対応しつつ、将来のUbuntu(X11)対応の余地を残した設計とします。また、プロジェクト管理ツールとして `uv` を使用し、Python 3.13環境を構築します。

## Approach
Pythonを用いて、OSのクリップボードとキーボード操作をエミュレートするアプローチを採用します。

1. **クリップボード経由のテキスト転送**: 文字化けや入力漏れを防ぐため、OSのクリップボードにテキストをコピーします。
2. **キーボードエミュレーション (ペースト操作)**: 現在アクティブになっているウィンドウ（つまり `claude` CLI が動いているターミナル）に対して、ペーストのショートカットキー（`Shift + Insert`）を送信します。
3. **Enterは送信しない**: ペースト操作のみを行い、改行コード(`\n`)や `Enter` キーのストロークは送信しないことで、ユーザーがプロンプト画面でテキストをレビュー・編集できる状態を保ちます。

## 実装ステップ

### Step 1: プロジェクトの初期化 (`uv` を使用)
新しいワークスペースディレクトリ `/c/src/claude-input` を作成し、`uv` を用いて初期化と仮想環境の作成を行います。
```bash
mkdir -p /c/src/claude-input
cd /c/src/claude-input
uv init --bare
uv venv -p 3.13
source .venv/Scripts/activate # Windows(Git Bash)の場合
uv add pyperclip pyautogui
```

### Step 2: コアロジック (Injectorクラス) の実装
OSの差異を吸収し、将来的に拡張しやすいように `src/injector.py` を作成します。今回は手動でウィンドウをアクティブにするため、指定秒数待機してからペースト（`Shift+Insert`）を実行します。

### Step 3: テスト用エントリポイント (CLI) の作成
手動で動作確認するためのシンプルなCLI `main.py` を作成します。コマンドライン引数からテキストと待機時間を受け取り、`injector.py` を呼び出します。

### Step 4: ドキュメント類の生成
新しいワークスペースで作業を引き継げるように、以下のドキュメントを生成します。
- `README.md`: プロジェクトの概要と実行方法
- `CLAUDE.md`: Claude Code用の開発ガイドライン
- `docs/plan.md`: 本計画の全体像
- `docs/design.md`: アーキテクチャ設計
- `docs/task.md`: 今後のタスクリスト（自動ウィンドウアクティブ化、Discord連携など）

## Verification (動作確認方法)
新しいワークスペースで `claude` を起動し、以下の手順で手動テストを行います。
1. `uv run main.py "テストメッセージ" --delay 3` を実行する。
2. 3秒のカウントダウンの間に、急いで別のターミナルで起動している `claude` の画面をクリックしてアクティブにする。
3. `claude` のプロンプトにテキストがペーストされ、Enterが押されずに待機状態になることを確認する。
