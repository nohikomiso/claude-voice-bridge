# アーキテクチャ設計 (Design)

## 概要
このプロジェクト `claude-input` は、Discordなどの外部ソースから受け取ったメッセージを、既存で動作している `claude` CLI（ターミナル）の入力プロンプトに対して、OSのGUIエミュレーションを通じて注入（ペースト）する仕組みを提供するものです。

## コア・コンポーネント

1. **TerminalInjector クラス (`injector.py`)**
   - **責務**: OSの差異（Windows/Ubuntu X11/Mac）を吸収し、クリップボード操作とキーボードエミュレーション（ペースト操作）を一元管理する。
   - **メソッド**:
     - `_prepare_clipboard(text)`: `pyperclip` を使用してテキストをOSのクリップボードに格納する。マルチバイト文字や長文の入力漏れを防ぐための安全策。
     - `_execute_paste()`: `pyautogui` を用いて、ターミナルで汎用的なペーストショートカットである `Shift + Insert` を対象ウィンドウに対して送信する。
     - `inject(text, delay_seconds)`: 上記処理をオーケストレーションし、人間が対象ウィンドウを手動でアクティブにするための猶予時間を提供するインターフェース。

2. **CLI エントリポイント (`main.py`)**
   - **責務**: ユーザーが手動でPoC（概念実証）をテストするためのコマンドラインインターフェースを提供する。
   - **依存関係**: `argparse` を用いてコマンドライン引数（注入するテキスト、待機時間）を受け取り、`TerminalInjector` を呼び出す。

## 将来の拡張設計 (Future Architecture)
現在のPoCは「待機時間中に人間が対象のターミナルをクリックする」という手動ステップに依存しています。
今後の拡張では、以下のコンポーネントを追加する想定です。

- **WindowLocator**: 対象となるターミナルウィンドウ（例: ウィンドウタイトルが `claude` または特定のターミナルクラス名）を特定し、プログラムからフォアグラウンドに引き上げる処理。
  - Windows: `pygetwindow` や `win32gui` を活用。
  - Ubuntu(X11): `wmctrl` や `xdotool` を活用。
- **DiscordAdapter**: Discord Botとして常駐し、特定のチャンネルに投稿されたメッセージをフックして `TerminalInjector` に渡す非同期リスナー。
